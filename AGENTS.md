# 项目协作规则

## 定位

“诗弦”是一个 TypeScript + Vite 的《琵琶行》互动长卷 Demo，以 Canvas 文字弦、Verlet 物理和 Web Audio 琵琶合成为核心。

## 常用命令

- `npm run dev`：本地开发，默认 `http://127.0.0.1:5173/`
- `npm run typecheck`：TypeScript 检查
- `npm run build`：类型检查并构建生产包

## 目录约定

- `src/content`：作品原文和内容模型，不依赖 DOM、Canvas、音频。
- `src/core`：可复用的物理、渲染、视口和音频能力，不写页面流程。
- `src/features/poem-stage`：作品分列、舞台尺寸、导航状态机、UI 呈现和交互编排。
- `src/themes`：作品旋律与声音映射。
- `docs`：现役架构、产品定义、内容模型和路线图。
- `outputs`：用户可交付截图与恢复备份；现有备份不得覆盖或删除。
- `work`：参考资料、历史快照和临时现场；清理前先列候选并取得确认。

## 当前行为

- 44 行正文完整展示，正文为 22 列，每列通常合并两句。
- 卷首、正文和曲终是同一连续空间；曲终结句固定两列。
- 初次进入停留在静止卷首；从右向左连续扫过题字左侧四根弦中的至少三根，才触发题字退场和后续自动播放。顶部播放按钮是无障碍替代入口。
- 抵住左右卷缘、拖动时间轴或使用方向键阅卷；卷轴从右侧卷首向左展开，滚轮和空白拖动不移动画卷。
- 顶部播放按钮以慢速自动展开长卷；暂停、时间轴和方向键必须能稳定接管当前位置。
- 卷首进入正文与曲终分别有独立琵琶衔接句；浏览器首次发声需要一次用户手势激活 Web Audio。
- 每列对应不同音高；横扫方向影响音色；点击列播放旋律短句。
- 改动交互后必须回归全文、曲终门槛、自动播放、声音、键盘、移动端和 reduced motion。
- 改动卷首后必须回归：静止等待、反方向不触发、三弦门槛、分弦音高、标题退场后自动播放，以及同一次拨弦手势不会误暂停播放。
- 改动方向交互后必须按阅读进度回归：卷轴滑块从右向左展开；左卷缘让 `progress` 增加并展示更多诗句，右卷缘让 `progress` 减少并返回卷首。必须同时验证画面、进度和曲终门槛。

## 下一步
